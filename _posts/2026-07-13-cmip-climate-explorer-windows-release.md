做气候变化、生态和遥感分析时，数据准备往往比计算本身更麻烦。CMIP6 文件分散在不同节点，变量和情景名称不够直观，下载后还要把 NetCDF 转成 GIS 软件常用的 GeoTIFF。

为减少这些重复操作，我制作了 **CMIP Climate Explorer**。它是一款面向 Windows 10/11 的中文桌面软件，主要完成两件事：**下载气候数据**，以及**把下载的数据整理成可以直接用于空间分析的 TIF**。安装版已经包含运行环境，不需要另外安装 Python。

## 它能做什么

### 1. 用中文查找气候变量

软件接入了 ESGF、Copernicus CDS、AWS Open Data、Microsoft Planetary Computer、NASA POWER、NASA Earthdata 和 NOAA NCEI。切换数据源后，可以继续选择变量、模型、情景、频率和年份。

常用变量已经增加中文名称。例如：

| 日常说法 | 常用变量 |
|---|---|
| 平均气温 | 近地表气温 `tas` |
| 最低气温 | 近地表最低气温 `tasmin` |
| 最高气温 | 近地表最高气温 `tasmax` |
| 降水量 | 降水通量 `pr` |

中文名称只用于帮助识别，软件检索时仍使用数据源规定的英文变量代码。如果 API 后续出现尚未翻译的新变量，列表会保留原英文名称，不会把它隐藏。

### 2. 下载连续多年数据

查询结果会尽量按变量、模型、情景和频率整理为长期序列。选择一个系列后，软件会把其中覆盖所选年份的文件加入同一个下载任务，不需要逐年点击。

下载任务页会显示状态、进度、速度和保存位置，并支持暂停、继续、取消、失败后重连和断点续传。已经下载完成的数据会归入本地数据，任务列表可以只保留仍需关注的项目。

### 3. 将NetCDF整理为GeoTIFF

文件处理页会按下载目录识别数据组，而不是要求逐个选择 NC 文件。软件会读取变量、模型、情景、时间范围和数据频率，再根据原始时间尺度提供可用的输出方式。

- 小时和日数据可以生成日、月或年 TIF。
- 月数据可以生成月或年 TIF，但不会反向生成并不存在的日数据。
- 年数据直接生成年度 TIF。
- 气温可自动由 K 转为摄氏度。
- 降水可根据时间间隔换算为 mm 累计量或平均强度。
- 月度和年度合成可选择平均、总和、最大、最小或众数等规则。

处理前仍应根据变量含义选择统计方法。气温通常使用平均值，累计降水通常使用总和；极端气候研究则可能需要最大值或最小值。

### 4. 重采样与研究区裁剪

输出时可以保持原始网格，也可以选择最近邻、双线性、平均值等重采样方法。目标网格既可以使用预设分辨率，也可以读取一个模板 TIF 的坐标系、像元大小和范围。

软件还支持读取 Shapefile、GeoPackage、GeoJSON、KML、GML 等矢量研究区。使用 Shapefile 时，需要把同名的 `.shp`、`.shx`、`.dbf` 和 `.prj` 等配套文件放在同一目录。完整处理顺序为：

```text
读取NetCDF -> 时间合成 -> 单位转换 -> 重采样 -> 按研究区裁剪 -> 输出GeoTIFF
```

中间文件会放在临时目录，任务成功、失败或主动停止后都会清理，最终目录只保留需要的结果。

## 最简单的使用流程

1. 从 GitHub Release 下载并安装软件，安装位置可以自行选择。
2. 在“设置”中确认数据保存目录。电脑有 D 盘时，软件优先建议使用 D 盘。
3. 在“数据下载”中选择数据源和变量，设置模型、情景与年份后查询并下载。
4. 下载完成后进入“文件处理”，选择数据组、输出时间尺度和处理参数，生成 TIF。

下载或处理过程中可以切换页面查看任务，但不要直接关闭程序。参数选错时可点击停止，并选择是否删除本次已经生成的结果。

## 数据源和账号说明

不同数据源的开放方式并不相同。ESGF、AWS Open Data、Planetary Computer 和 NASA POWER 中的公开数据可以由软件直接查询或下载；Copernicus CDS 和部分 NASA Earthdata 数据需要用户在官方网站注册、同意许可或完成授权。

软件不会绕过数据提供方的访问规则。遇到需要账号的数据时，会明确显示授权要求或官方入口，而不是让任务长时间停在 0%。

## 存储、更新与开源

安装目录和气候数据目录是两回事。程序可以安装在任意磁盘，大体积的 NetCDF 与 GeoTIFF 则保存在“设置”中选择的数据目录。这样可以避免系统盘空间被气候文件占满。

设置页提供“检查更新”。软件会读取 GitHub 上的最新正式版，下载 Windows 安装包并校验 SHA-256；只有校验一致才会启动更新安装。源代码、版本记录和安装包均可在项目仓库中查看。

CMIP Climate Explorer 仍在持续完善。气候数据文件体积大、节点状态和网络环境差异明显，正式分析前仍应检查变量单位、时间覆盖、坐标系、缺失值以及合成结果是否符合研究设计。
